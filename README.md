# 微信文章抓取器 · Chrome 浏览器插件 v0.4.0

一键抓取微信公众号文章，保存为 Markdown + 图片到本地，支持同步至飞书云文档。

---

## ✨ 功能

- **📥 一键抓取**：自动提取文章标题、作者、正文和所有配图
- **📝 Markdown 转换**：自研 HTML→Markdown 引擎，零依赖
- **🖼️ 图片下载**：自动下载所有配图
- **📦 ZIP 打包**：文章 + 图片 + 元数据打包下载
- **📋 批量队列**：在工作台粘贴多篇文章链接，后台逐条打开、抓取、保存和同步；完成后可打开/复制飞书链接、本地路径，并导出 CSV
- **☁️ 飞书同步双通道**：
  - 🖥️ **通道 1**：自动检测本地 `lark-cli` 服务，有就用（零配置）
  - 🔑 **通道 2**：飞书 API 直连，配置 App ID/Secret 即可（无需安装任何东西）
- **📍 飞书保存位置**：可保存到默认位置、我的文档库、飞书文件夹或知识库节点

## 🚀 安装

1. 打开 `chrome://extensions/` → 开启「开发者模式」
2. 点击「加载已解压的扩展程序」→ 选择 `wechat-capture-extension/` 文件夹
3. 工具栏出现绿色图标，安装完成

## 📖 使用

1. 打开任意微信公众号文章
2. 点击插件图标 →「📥 抓取文章」
3. ZIP 自动下载到本地
4. [可选] 点击「☁️ 同步到飞书」

同步到飞书时，首页会显示一行「保存到」。点击「更换」后粘贴飞书文件夹链接、知识库节点链接，或填写 `my_library` 保存到我的文档库。
成功保存过的位置会保存在浏览器本地，最多保留 5 个，之后可在首页一键切换。

插件弹窗只保留快捷抓取。点击「工作台」进入完整管理页：批量队列、本地设置、飞书同步和通知都在同一个页面里。批量队列有独立的本次运行配置，默认保存到本地；开始前可以切换为飞书或两者，并指定本地目录、飞书保存位置和是否同步图片。批量结果支持导出 CSV，字段已预置，也可以拖拽调整字段顺序和选择是否导出。

## ☁️ 飞书同步 — 双通道说明

插件会**自动选择**可用通道：

| 通道 | 条件 | 需要什么 |
|------|------|---------|
| **lark-cli** | 本地运行了 `companion/feishu_sync_server.py` | 已安装 `lark-cli` + 已登录飞书 |
| **API 直连** | 未检测到 lark-cli 服务 | 在设置中填入飞书 App ID + App Secret |

### 通道 1：lark-cli（自用推荐）

```bash
cd companion
python3 feishu_sync_server.py
```

如果已经安装过后台服务，更新插件代码后重新运行：

```bash
cd companion
bash install.sh
```

Chrome 扩展不能直接启动本机进程。`install.sh` 会把 companion 注册为 macOS `launchd` 服务，包含 `RunAtLoad` 和 `KeepAlive`，因此安装一次后会在登录时自动启动，异常退出也会自动拉起。

### 通道 2：API 直连（给别人用）

1. 前往 [飞书开放平台](https://open.feishu.cn/app) 创建企业自建应用
2. 获取 App ID 和 App Secret
3. 在插件设置页面填入，点击「测试连接」
4. 配置完成，无需任何本地服务

> 保存到指定飞书位置需要对应位置的写入权限。`lark-cli` 通道会优先用用户身份检测位置；API 直连会在同步时校验写入权限。
